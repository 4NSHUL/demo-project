# Complaint Register Demo Pitch Deck Design

## Purpose

Give hackathon judges a concise, credible story that leads naturally into a live demo of the Complaint Register.

## Audience outcome

By the end, hackathon judges should understand that the Complaint Register makes community issues visible and accountable from submission through resolution.

## Scope

Create a six-slide PowerPoint deck with a minimal civic-modern visual treatment:

1. Product promise: a clearer way for communities to act on issues.
2. Problem: issues disappear across calls, chats, and follow-ups.
3. Solution: a shared, searchable, accountable register.
4. Live flow: submit, triage, update, resolve.
5. Credibility: responsive UI, browser-local demo mode, optional shared Supabase mode.
6. Close: make every community issue visible until it is resolved.

## Constraints

- Use only claims supported by the current application and README.
- Keep visible copy sparse and audience-facing.
- Use a 16:9 PowerPoint format with large, legible typography.
- Do not alter application code.

## Verification

Render each slide, inspect the final deck at full size, and run the slide overflow check before delivery.
